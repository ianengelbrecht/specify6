Interactions

Default settings for fields on the Loan and other Interactions forms can be set in the Interactions Preference. New Interactions will include the default values, which are editable.

  • For Mac: Choose Specify > Preferences > Interactions.
  • For Windows and Linux: Choose Edit > Preferences > Interactions.

 

Interactions Preference

 

  • Due in Months can be set between one and 60 months.

  • Shipment Method choices are:
    • By Hand
    • DHL
    • Fed Ex
    • UPS
    • USPS

  • The Shipment Method list is a System Pick List and can be edited using the pick list editor.

  • Shipped By is a Query Combo Box to the agent table. This allows data to be searched and chosen from the agent table, not typed directly into the box. It is possible to type data directly into the box, but the box will not accept the data; the data must first be added to the table by clicking the  (add) button and creating a new record.

    Search for an existing agent

    Using a Query Combo Box

    1. Type the first letter, or first few letters of the name in the box, then either click on the button or use the down-arrow key on the keyboard. A list of agent matching the typed letter(s) will appear.

    If Specify can not find a match based on the typed letters then an  button will appear. Click the button or press the enter key to add a new record.

    If the required name is not in the resulting list, either press the enter key when 'Add ...' is in focus, or click the  (add) button, to add a new record.

    Pressing the esc (escape) key will cancel the list box.

    Note: Type more letters into the box to reduce the results and expedite the search.

    Using the Search button (these buttons are not available with keystroke entry):

    1. Click on the (Search) button. A search dialog will appear.
    2. Type in either the entire name or the first few letters of the search followed by an asterisk (*).
    3. Select a record from the results.

    Add a new agent:

    1. Click on the (add) button and enter data into the resulting dialog.

     

    Edit an existing agent record:

    1. Choose a record from the list.
    2. Click the (edit) icon to open the table in an editable dialog.

     

Default Source Table allows users to choose the source for their interactions based on the appropriate table (Collection Object or Preparations) and field with an identifier. For instance, if preparations in the collection include an identifier and the user wishes to reference that identifier then they will choose 'Preparation' as the Default Source for the Table.

Note: Interactions are scoped to the Discipline level and therefore the settings in the Interactions Preference will affect all other users in the Collection and Discipline. If collections within the Discipline use different tables this setting should be set to 'Ask'.

  • Table offers the following choices:
    • Ask - indicates that either the Collection Object or Preparations table may be used in Interactions and allows the user to choose the appropriate table at the time that the interaction is created
    • Collection Object - indicates that the Collection Object table is the only one used for Interactions for the Collections in the Discipline.
    • Preparation - indicates that the Preparation table is the only one used for Interactions for the Collections in the Discipline.

Item Identifier Fields allows users to choose the appropriate field for the identifier based on the Default Source Table choice and database setup. 

  • Collection Object table has four fields available to use as an identifier field but fields that are not used and hidden they will not be presented as choices. If the field you require is not shown as a choice you can Unhide the field using the Schema Configuration tool. The most common fields are shown below:
    • Alt Cat Number
    • Catalog Number
    • GUID
    • Project Number

Other possible fields for IDs in the Collection Object table include Bar Code, Modifier, Name, Project Number, Text1, Text2 and Text 3. Catalog Number and GUID fields are forced to be unique in the system

  • Preparation table has two fields available to use as a unique identifier field but fields that are not used and hidden they will not be presented as choices. If the field you require is not shown as a choice you can unhide the field using the Schema Configuration tool.
    • Barcode
    • GUID

Other possible fields for IDs in the Preparation table include Sample Number, Text1, Text2 and Text 3, Text 4 ,Text 5, Text 6, Text 7, Text 8, Text 9. Barcode and GUID are forced to be unique in the system.
